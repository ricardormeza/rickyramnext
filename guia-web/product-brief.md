# Home + Oferta de Sitios (Next.js) — Estructura, Analogía, Paquetes, Add-ons, Brief y Sitemap

## 1) Analogía simple (Astro vs WordPress vs Next.js)

Piensa en tu presencia digital como **tres tipos de construcción**:

### Astro = “Casa ligera prefabricada de lujo”
- Súper rápida, elegante, barata de mantener.
- Ideal para sitios de marketing: landing, servicios, portafolio.
- **Cuando quieres velocidad, SEO y cero complicaciones.**

### WordPress = “Casa modular con cuarto de herramientas”
- Muy fácil de administrar (blog, páginas, contenido).
- Se adapta con plugins, pero si crece mucho puede volverse pesada.
- **Cuando necesitas publicar contenido seguido y un panel sencillo para el equipo.**

### Next.js = “Edificio a medida con ingeniería”
- Lo diseñas como tú quieras: web avanzada, dashboards, apps, e-commerce headless, integraciones.
- Escalabilidad + performance + UX premium.
- **Cuando necesitas funcionalidades especiales y crecimiento serio.**

## 4) Qué incluye “SEO básico” (para planes Starter)

- Titles y metas por página
- Estructura H1/H2 correcta
- Sitemap.xml y robots.txt
- Open Graph (OG) básico
- Schema básico (Organization / LocalBusiness)
- Optimización básica de performance (imágenes y carga)
- Checklist de indexación (Search Console cuando aplique)

> No incluye: estrategia de keywords amplia, linkbuilding, contenido mensual, ni tracking avanzado.

## 7) Product Brief (brief del proyecto del sitio en Next.js)

### 7.1 Para quién es el servicio (audiencia objetivo)
- Emprendedores y negocios locales que necesitan “presencia que convierta”.
- PyMEs que requieren un sitio profesional para captar leads y vender servicios.
- Empresas que buscan una web moderna (performance, UX, escalabilidad) y opciones de crecimiento (SEO/Ads/automatización).
- Leads “comparadores” que no saben qué tecnología elegir y necesitan guía clara.

### 7.2 Meta del sitio (objetivo principal)
- **Convertir visitas en leads** (WhatsApp, formulario, llamada, agendar).
- **Educar rápido** al usuario sobre diferencias Astro/WordPress/Next.js con una analogía sencilla.
- Presentar una **oferta productizada**: planes claros, alcances claros, upsells claros.
- Servir como hub para:
  - Portafolio / casos
  - Blog (WordPress headless)
  - Servicios recurrentes (SEO, Ads, mantenimiento, etc.)

### 7.3 Alcance del sitio (por ahora)
- Sitio **full responsive** (mobile-first).
- No se define aún el diseño final ni el contenido detallado por sección.
- La prioridad es definir **arquitectura, secciones, propuesta y estructura** lista para diseñar.

### 7.5 Tech stack (propuesto)
- **Framework:** Next.js (App Router)
- **UI:** Tailwind CSS + **shadcn/ui**
- **Componentes extra recomendados (elige según necesidades):**
  - **Framer Motion** (animaciones suaves, microinteracciones)
  - **Lucide Icons** (iconografía consistente)
  - **next-seo** o manejo SEO manual con `metadata` (según preferencia)
  - **Zod + React Hook Form** (formularios y validación)
  - **Resend** o **SendGrid** (envío de formularios por email)
  - **Cal.com** (o Calendly embed) para “Agendar”
- **CMS para blog:** WordPress Headless
  - Consumo por REST API o GraphQL (si usas WPGraphQL)
- **Hosting sugerido:** Vercel (por sinergia con Next.js)
- **Analítica:** (opcional por plan) GA4 / GTM / pixels

### 7.6 Consideraciones clave
- Enfoque de performance: imágenes optimizadas, fuentes, lazy loading.
- Accesibilidad básica: navegación y contraste (a definir en diseño).
- SEO técnico desde el inicio (metadata, sitemap, estructura semántica).
- Preparar el sitio para crecer: servicios recurrentes y upsells integrados en UX.
